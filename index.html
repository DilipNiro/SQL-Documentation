Introduction au SQL

Le SQL (Structured Query Language) est un langage informatique spécialement conçu pour interagir avec les bases de données relationnelles. Créé dans les années 1970, il est devenu un standard de facto dans la gestion des données au sein de nombreuses applications et systèmes informatiques.

Présentation générale

Le SQL permet de manipuler les données stockées dans des bases de données de manière structurée et efficace. Il offre un ensemble complet de commandes pour effectuer diverses opérations sur les données.

Structure des données

Les bases de données relationnelles sont constituées de tables organisées en colonnes et lignes, particulièrement bien adaptées à l'utilisation du SQL.

Systèmes compatibles

De nombreux systèmes de gestion de bases de données (SGBD) prennent en charge le SQL :

  • MySQL
  • PostgreSQL
  • Microsoft SQL Server
  • Oracle
  • SQLite

Histoire et évolution

1970

Développement initial du SQL par IBM Research. Le langage était initialement appelé SEQUEL (Structured English Query Language).

1974

Publication des premiers articles décrivant le langage SQL.

1986

ANSI publie la première norme SQL (SQL-86).

1989

ISO adopte SQL comme norme internationale.

Aujourd'hui

Le SQL continue d'évoluer avec de nouvelles fonctionnalités et améliorations régulières.

Définitions

SQL (Structured Query Language)

Le SQL est un langage informatique conçu pour gérer et interagir avec les bases de données relationnelles. Il permet de créer, mettre à jour, supprimer et interroger des données dans une base de données.

Base de données

Une base de données est un ensemble organisé de données stockées électroniquement. Les données sont structurées de manière à pouvoir être facilement gérées, mises à jour et consultées.

SGBD

Un Système de Gestion de Base de Données est un logiciel qui permet de stocker, d'organiser et de gérer efficacement les données dans une base de données.

Table

Une table est une structure qui stocke des données sous forme de lignes (enregistrements) et de colonnes (champs). Chaque ligne représente un enregistrement unique, tandis que chaque colonne correspond à un attribut spécifique.

Requête

Une requête SQL est une instruction ou une commande qui permet d'interroger une base de données pour récupérer des données spécifiques. Les requêtes SQL sont utilisées pour extraire des informations, effectuer des calculs et générer des rapports à partir des données stockées.

-- Exemple de requête simple :
SELECT nom, prenom FROM utilisateurs WHERE age > 18;

Types de Requêtes

  • Requêtes de sélection (SELECT)
  • Requêtes de modification (UPDATE)
  • Requêtes d'insertion (INSERT)
  • Requêtes de suppression (DELETE)
  • Requêtes de création de structure (CREATE)

Commandes DDL (Data Definition Language)

Les commandes DDL sont utilisées pour définir et modifier la structure de la base de données. Ces commandes permettent de créer, modifier et supprimer les objets de la base de données.

Principales commandes DDL

CREATE TABLE Permet de créer une nouvelle table dans la base de données
ALTER TABLE Modifie la structure d'une table existante (ajouter/supprimer des colonnes, modifier des contraintes)
DROP TABLE Supprime complètement une table et toutes ses données
CREATE DATABASE Crée une nouvelle base de données
ALTER DATABASE Modifie les propriétés d'une base de données existante
DROP DATABASE Supprime complètement une base de données
CREATE INDEX Crée un index sur une ou plusieurs colonnes pour optimiser les recherches
DROP INDEX Supprime un index existant

Commandes DML (Data Manipulation Language)

Les commandes DML sont utilisées pour manipuler les données stockées dans une base de données.

Principales commandes DML

SELECT Récupère des données depuis une ou plusieurs tables en spécifiant les colonnes à récupérer et les conditions de filtrage
INSERT INTO Ajoute de nouveaux enregistrements dans une table
UPDATE Modifie des données existantes dans une table en fonction de certaines conditions
DELETE FROM Supprime des enregistrements d'une table en fonction de certaines conditions
MERGE Effectue une opération conditionnelle qui permet d'insérer, mettre à jour ou supprimer des données en fonction de certaines conditions
TRUNCATE TABLE Supprime toutes les données d'une table, mais conserve la structure de la table
REPLACE INTO Insère une nouvelle ligne dans une table ou remplace une ligne existante si une contrainte d'unicité est violée
UPSERT Une opération qui insère une ligne si elle n'existe pas ou la met à jour si elle existe déjà
CALL Utilisée pour exécuter des procédures stockées ou des fonctions définies par l'utilisateur

Commandes DCL (Data Control Language)

Les commandes DCL sont utilisées pour gérer les autorisations, les privilèges et les permissions d'accès aux données dans une base de données.

Principales commandes DCL

GRANT La commande GRANT est utilisée pour accorder des privilèges ou des autorisations à un utilisateur ou à un rôle sur des objets de la base de données, tels que des tables, des vues, des procédures stockées, etc.
REVOKE La commande REVOKE est utilisée pour révoquer des privilèges ou des autorisations précédemment accordés à un utilisateur ou à un rôle
DENY La commande DENY est utilisée pour refuser spécifiquement un privilège ou une autorisation à un utilisateur ou à un rôle, ce qui prévaut sur les autorisations accordées par GRANT
COMMIT La commande COMMIT est utilisée pour valider et enregistrer définitivement les modifications apportées à la base de données, les rendant ainsi permanentes
ROLLBACK La commande ROLLBACK est utilisée pour annuler et annihiler toutes les modifications non encore validées effectuées depuis le dernier COMMIT
SAVEPOINT La commande SAVEPOINT est utilisée pour définir un point de sauvegarde dans une transaction, ce qui permet de revenir à ce point si nécessaire sans annuler l'intégralité de la transaction

Les Objets SQL

En SQL, les "objets" font référence aux entités que vous manipulez dans une base de données. Les principaux types d'objets dans une base de données relationnelle incluent les tables, les vues, les procédures stockées, les déclencheurs et les index.

Principaux types d'objets

Table Une table est une structure fondamentale dans une base de données relationnelle. Elle est utilisée pour stocker des données sous forme de lignes et de colonnes. Chaque colonne a un nom et un type de données spécifique, tandis que chaque ligne représente un enregistrement unique. Les tables sont définies à l'aide de la commande DDL CREATE TABLE.
Vue Une vue est une représentation virtuelle d'une ou plusieurs tables dans une base de données. Elle permet de définir une requête SQL qui renvoie un ensemble de données spécifique. Les vues sont utiles pour simplifier l'accès aux données en fournissant un moyen de masquer la complexité des requêtes sous-jacentes.
Procédure stockée Une procédure stockée est un ensemble d'instructions SQL pré-enregistrées qui peuvent être exécutées en tant que routine. Elles sont utilisées pour effectuer des tâches spécifiques, telles que la manipulation de données, la génération de rapports ou d'autres opérations complexes. Les procédures stockées sont créées à l'aide de la commande DDL CREATE PROCEDURE.
Déclencheur (Trigger) Un déclencheur est un ensemble de code SQL associé à une table qui est automatiquement activé en réponse à certaines opérations, telles que l'insertion, la mise à jour ou la suppression de données dans la table. Les déclencheurs sont utiles pour appliquer des règles métier, effectuer des vérifications de contrainte ou enregistrer des audits.
Index Un index est une structure de données qui améliore les performances des opérations de recherche dans une table. Il est créé sur une ou plusieurs colonnes de la table et stocke des données permettant d'accéder rapidement aux enregistrements. Les index sont créés à l'aide de la commande DDL CREATE INDEX.

Créer une base de données

Syntaxe de base

Syntaxe simple CREATE DATABASE nom_de_la_base_de_donnees;
Avec options (MySQL) CREATE DATABASE MaBaseDeDonnees CHARACTER SET utf8 COLLATE utf8_general_ci;

Création de table

Syntaxe générale CREATE TABLE nom_de_la_table ( nom_colonne1 type_de_donne1, nom_colonne2 type_de_donne2, ... nom_colonneN type_de_donneN );
Exemple concret CREATE TABLE utilisateurs ( id INT PRIMARY KEY, nom VARCHAR(50), prenom VARCHAR(50), age INT, email VARCHAR(100), date_inscription DATE );
