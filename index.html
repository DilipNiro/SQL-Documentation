Introduction au SQL

Le SQL (Structured Query Language) est un langage informatique spécialement conçu pour interagir avec les bases de données relationnelles. Créé dans les années 1970, il est devenu un standard de facto dans la gestion des données au sein de nombreuses applications et systèmes informatiques.

Présentation générale

Le SQL permet de manipuler les données stockées dans des bases de données de manière structurée et efficace. Il offre un ensemble complet de commandes pour effectuer diverses opérations sur les données.

Structure des données

Les bases de données relationnelles sont constituées de tables organisées en colonnes et lignes, particulièrement bien adaptées à l'utilisation du SQL.

Systèmes compatibles

De nombreux systèmes de gestion de bases de données (SGBD) prennent en charge le SQL :

  • MySQL
  • PostgreSQL
  • Microsoft SQL Server
  • Oracle
  • SQLite

Histoire et évolution

1970

Développement initial du SQL par IBM Research. Le langage était initialement appelé SEQUEL (Structured English Query Language).

1974

Publication des premiers articles décrivant le langage SQL.

1986

ANSI publie la première norme SQL (SQL-86).

1989

ISO adopte SQL comme norme internationale.

Aujourd'hui

Le SQL continue d'évoluer avec de nouvelles fonctionnalités et améliorations régulières.

Définitions

SQL (Structured Query Language)

Le SQL est un langage informatique conçu pour gérer et interagir avec les bases de données relationnelles. Il permet de créer, mettre à jour, supprimer et interroger des données dans une base de données.

Base de données

Une base de données est un ensemble organisé de données stockées électroniquement. Les données sont structurées de manière à pouvoir être facilement gérées, mises à jour et consultées.

SGBD

Un Système de Gestion de Base de Données est un logiciel qui permet de stocker, d'organiser et de gérer efficacement les données dans une base de données.

Table

Une table est une structure qui stocke des données sous forme de lignes (enregistrements) et de colonnes (champs). Chaque ligne représente un enregistrement unique, tandis que chaque colonne correspond à un attribut spécifique.

Requête

Une requête SQL est une instruction ou une commande qui permet d'interroger une base de données pour récupérer des données spécifiques. Les requêtes SQL sont utilisées pour extraire des informations, effectuer des calculs et générer des rapports à partir des données stockées.

-- Exemple de requête simple :
SELECT nom, prenom FROM utilisateurs WHERE age > 18;

Types de Requêtes

  • Requêtes de sélection (SELECT)
  • Requêtes de modification (UPDATE)
  • Requêtes d'insertion (INSERT)
  • Requêtes de suppression (DELETE)
  • Requêtes de création de structure (CREATE)

Commandes DDL (Data Definition Language)

Les commandes DDL sont utilisées pour définir et modifier la structure de la base de données. Ces commandes permettent de créer, modifier et supprimer les objets de la base de données.

Principales commandes DDL

CREATE TABLE Permet de créer une nouvelle table dans la base de données
ALTER TABLE Modifie la structure d'une table existante (ajouter/supprimer des colonnes, modifier des contraintes)
DROP TABLE Supprime complètement une table et toutes ses données
CREATE DATABASE Crée une nouvelle base de données
ALTER DATABASE Modifie les propriétés d'une base de données existante
DROP DATABASE Supprime complètement une base de données
CREATE INDEX Crée un index sur une ou plusieurs colonnes pour optimiser les recherches
DROP INDEX Supprime un index existant

Commandes DML (Data Manipulation Language)

Les commandes DML sont utilisées pour manipuler les données stockées dans une base de données.

Principales commandes DML

SELECT Récupère des données depuis une ou plusieurs tables en spécifiant les colonnes à récupérer et les conditions de filtrage
INSERT INTO Ajoute de nouveaux enregistrements dans une table
UPDATE Modifie des données existantes dans une table en fonction de certaines conditions
DELETE FROM Supprime des enregistrements d'une table en fonction de certaines conditions
MERGE Effectue une opération conditionnelle qui permet d'insérer, mettre à jour ou supprimer des données en fonction de certaines conditions
TRUNCATE TABLE Supprime toutes les données d'une table, mais conserve la structure de la table
REPLACE INTO Insère une nouvelle ligne dans une table ou remplace une ligne existante si une contrainte d'unicité est violée
UPSERT Une opération qui insère une ligne si elle n'existe pas ou la met à jour si elle existe déjà
CALL Utilisée pour exécuter des procédures stockées ou des fonctions définies par l'utilisateur

Commandes DCL (Data Control Language)

Les commandes DCL sont utilisées pour gérer les autorisations, les privilèges et les permissions d'accès aux données dans une base de données.

Principales commandes DCL

GRANT La commande GRANT est utilisée pour accorder des privilèges ou des autorisations à un utilisateur ou à un rôle sur des objets de la base de données, tels que des tables, des vues, des procédures stockées, etc.
REVOKE La commande REVOKE est utilisée pour révoquer des privilèges ou des autorisations précédemment accordés à un utilisateur ou à un rôle
DENY La commande DENY est utilisée pour refuser spécifiquement un privilège ou une autorisation à un utilisateur ou à un rôle, ce qui prévaut sur les autorisations accordées par GRANT
COMMIT La commande COMMIT est utilisée pour valider et enregistrer définitivement les modifications apportées à la base de données, les rendant ainsi permanentes
ROLLBACK La commande ROLLBACK est utilisée pour annuler et annihiler toutes les modifications non encore validées effectuées depuis le dernier COMMIT
SAVEPOINT La commande SAVEPOINT est utilisée pour définir un point de sauvegarde dans une transaction, ce qui permet de revenir à ce point si nécessaire sans annuler l'intégralité de la transaction

Les Objets SQL

En SQL, les "objets" font référence aux entités que vous manipulez dans une base de données. Les principaux types d'objets dans une base de données relationnelle incluent les tables, les vues, les procédures stockées, les déclencheurs et les index.

Principaux types d'objets

Table Une table est une structure fondamentale dans une base de données relationnelle. Elle est utilisée pour stocker des données sous forme de lignes et de colonnes. Chaque colonne a un nom et un type de données spécifique, tandis que chaque ligne représente un enregistrement unique. Les tables sont définies à l'aide de la commande DDL CREATE TABLE.
Vue Une vue est une représentation virtuelle d'une ou plusieurs tables dans une base de données. Elle permet de définir une requête SQL qui renvoie un ensemble de données spécifique. Les vues sont utiles pour simplifier l'accès aux données en fournissant un moyen de masquer la complexité des requêtes sous-jacentes.
Procédure stockée Une procédure stockée est un ensemble d'instructions SQL pré-enregistrées qui peuvent être exécutées en tant que routine. Elles sont utilisées pour effectuer des tâches spécifiques, telles que la manipulation de données, la génération de rapports ou d'autres opérations complexes. Les procédures stockées sont créées à l'aide de la commande DDL CREATE PROCEDURE.
Déclencheur (Trigger) Un déclencheur est un ensemble de code SQL associé à une table qui est automatiquement activé en réponse à certaines opérations, telles que l'insertion, la mise à jour ou la suppression de données dans la table. Les déclencheurs sont utiles pour appliquer des règles métier, effectuer des vérifications de contrainte ou enregistrer des audits.
Index Un index est une structure de données qui améliore les performances des opérations de recherche dans une table. Il est créé sur une ou plusieurs colonnes de la table et stocke des données permettant d'accéder rapidement aux enregistrements. Les index sont créés à l'aide de la commande DDL CREATE INDEX.

Créer une Base de Données (BDD)

Syntaxe de base

Pour créer une base de données (BDD) en SQL, on utilise la commande DDL (Data Definition Language) CREATE DATABASE. Cette commande permet de définir une nouvelle base de données, en spécifiant des options de configuration si nécessaire.

Syntaxe simple CREATE DATABASE nom_de_la_base_de_donnees;
Avec options (MySQL) CREATE DATABASE MaBaseDeDonnees CHARACTER SET utf8 COLLATE utf8_general_ci;

Explications et Options

La syntaxe peut légèrement différer selon le système de gestion de base de données (SGBD) utilisé. Par exemple, dans MySQL, des options comme le jeu de caractères (CHARACTER SET) et le classement (COLLATE) permettent de spécifier les paramètres de codage de la base de données. Cela est utile pour adapter la base de données aux langues et aux besoins spécifiques des utilisateurs.

Exemples d'utilisation

Création d'une base de données simple CREATE DATABASE EntrepriseDB;
Création avec paramètres de langue (MySQL) CREATE DATABASE MonSiteDB CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

Créer une Table

Syntaxe de base

Pour créer une table dans une base de données SQL, vous utilisez la commande DDL (Data Definition Language) CREATE TABLE. Cette commande permet de définir une table en précisant le nom, les colonnes, les types de données associés, et les contraintes.

Syntaxe générale CREATE TABLE nom_de_la_table ( nom_colonne1 type_de_donne1, nom_colonne2 type_de_donne2, ... nom_colonneN type_de_donneN );

Exemple Concret

L'exemple suivant montre comment créer une table nommée utilisateurs avec des colonnes représentant des informations sur chaque utilisateur.

Exemple d'une table "utilisateurs" CREATE TABLE utilisateurs ( id INT PRIMARY KEY, nom VARCHAR(50), prenom VARCHAR(50), age INT, email VARCHAR(100), date_inscription DATE );

Explication des Colonnes et Contraintes

- id: Défini comme clé primaire (PRIMARY KEY), garantissant l'unicité de chaque enregistrement.
- nom, prenom, email: Colonnes de type VARCHAR pour stocker des chaînes de caractères, avec une longueur maximale spécifiée entre parenthèses.
- age: Colonne de type INT pour stocker l'âge en entier.
- date_inscription: Colonne de type DATE pour enregistrer la date d'inscription de chaque utilisateur.

Définir des Contraintes

Vous pouvez ajouter différentes contraintes pour assurer l'intégrité des données, telles que :

  • PRIMARY KEY: Identifie de manière unique chaque ligne dans la table.
  • NOT NULL: Empêche les valeurs nulles dans une colonne.
  • UNIQUE: Garantit que toutes les valeurs d'une colonne sont uniques.
  • FOREIGN KEY: Crée une relation entre deux tables.
  • CHECK: Applique une contrainte pour limiter les valeurs d'une colonne.
  • DEFAULT: Définit une valeur par défaut si aucune valeur n'est spécifiée.

Exemple de Table avec Contraintes

Exemple avec clés et contraintes CREATE TABLE employes ( id INT PRIMARY KEY, nom VARCHAR(50) NOT NULL, prenom VARCHAR(50) NOT NULL, age INT CHECK (age >= 18), email VARCHAR(100) UNIQUE, departement_id INT, FOREIGN KEY (departement_id) REFERENCES departements(id) );

INSERT / UPDATE / DELETE / DROP

Introduction

Les commandes INSERT, UPDATE, DELETE et DROP en SQL sont essentielles pour la manipulation des données et des structures de tables dans une base de données. Ces commandes permettent d'ajouter, de modifier, de supprimer des enregistrements et des objets de la base de données.

INSERT INTO

La commande INSERT INTO est utilisée pour ajouter de nouveaux enregistrements (lignes) dans une table existante. Vous spécifiez la table dans laquelle insérer les données, ainsi que les colonnes et les valeurs à insérer.

Syntaxe INSERT INTO nom_de_la_table (colonne1, colonne2, ...) VALUES (valeur1, valeur2, ...);
Exemple INSERT INTO utilisateurs (nom, prenom, age, email) VALUES ('Dupont', 'Jean', 30, 'jean.dupont@example.com');

UPDATE

La commande UPDATE modifie les données existantes dans une table. Elle permet de spécifier les colonnes à mettre à jour, les nouvelles valeurs, et les conditions pour cibler des enregistrements spécifiques.

Syntaxe UPDATE nom_de_la_table SET colonne1 = nouvelle_valeur1, colonne2 = nouvelle_valeur2, ... WHERE condition;
Exemple UPDATE utilisateurs SET email = 'nouveau.email@example.com' WHERE id = 1;

DELETE FROM

La commande DELETE FROM supprime des enregistrements d'une table selon une condition donnée. Attention : si aucune condition n'est spécifiée, toutes les lignes de la table seront supprimées.

Syntaxe DELETE FROM nom_de_la_table WHERE condition;
Exemple DELETE FROM utilisateurs WHERE id = 1;

DROP

La commande DROP est utilisée pour supprimer des objets de la base de données, tels que des tables, des vues, des procédures stockées, des déclencheurs, des index, ou même des bases de données entières. Attention : l'utilisation de DROP est irréversible et entraîne la perte complète de l'objet.

Syntaxe (Table) DROP TABLE nom_de_la_table;
Syntaxe (Base de données) DROP DATABASE nom_de_la_base_de_donnees;

Clés Primaires et Clés Étrangères

Clé Primaire

Une clé primaire est un concept fondamental dans une base de données relationnelle. Elle est utilisée pour identifier de manière unique chaque enregistrement (ligne) dans une table. Voici quelques caractéristiques essentielles de la clé primaire :

Caractéristiques de la clé primaire :
- Unicité : Chaque valeur dans la colonne définie comme clé primaire doit être unique dans la table, garantissant ainsi qu'aucune autre ligne ne partage la même valeur.
- Obligation : Les valeurs dans la colonne de clé primaire doivent être non nulles (pas de valeurs nulles autorisées).
- Identification unique : La clé primaire identifie chaque enregistrement de façon unique, assurant qu'il n'existe pas de doublons dans la table.
- Indexation : Un index est souvent créé automatiquement sur la colonne de clé primaire pour optimiser les performances des recherches et des jointures.
- Références : Les clés primaires sont fréquemment utilisées pour établir des relations avec d'autres tables en tant que clés étrangères, permettant ainsi de relier les données entre les tables.

Définition de la Clé Primaire

Pour définir une clé primaire lors de la création d'une table, utilisez la clause PRIMARY KEY après la déclaration de la colonne.

Exemple CREATE TABLE employes ( id INT PRIMARY KEY, nom VARCHAR(50), prenom VARCHAR(50), email VARCHAR(100) );

Clé Étrangère

Une clé étrangère est un élément clé de la modélisation des bases de données relationnelles, servant à établir des liens entre tables. Elle établit une relation entre la table "enfant" (qui contient la clé étrangère) et la table "parente" (référencée). Les clés étrangères garantissent l'intégrité référentielle, assurant ainsi que les données sont cohérentes entre les tables liées.

Caractéristiques de la clé étrangère :
- Relation entre les tables : La clé étrangère relie une colonne de la table enfant à une colonne de clé primaire dans la table parente, spécifiant comment les données sont liées entre elles.
- Intégrité référentielle : La clé étrangère maintient l'intégrité référentielle, c'est-à-dire que les valeurs dans la table enfant doivent obligatoirement exister dans la table parente, évitant ainsi les données incohérentes.
- Opérations en cascade : Des actions en cascade peuvent être configurées pour qu'une mise à jour ou une suppression dans la table parente entraîne automatiquement des mises à jour ou suppressions correspondantes dans la table enfant.
- Indexation : Un index peut être créé sur la colonne de clé étrangère pour optimiser les opérations de jointure entre les tables.

Définition de la Clé Étrangère

Pour définir une clé étrangère, utilisez la clause FOREIGN KEY suivie de la table et de la colonne de référence dans la table parente.

Exemple CREATE TABLE departements ( id INT PRIMARY KEY, nom VARCHAR(100) ); CREATE TABLE employes ( id INT PRIMARY KEY, nom VARCHAR(50), prenom VARCHAR(50), departement_id INT, FOREIGN KEY (departement_id) REFERENCES departements(id) );

Actions en Cascade avec Clés Étrangères

Les clés étrangères peuvent inclure des actions en cascade pour spécifier ce qui doit se produire lorsque les données dans la table parente sont modifiées ou supprimées. Les options les plus utilisées sont ON DELETE CASCADE et ON UPDATE CASCADE.

Explication des options en cascade :
- ON DELETE CASCADE : Supprime automatiquement les enregistrements liés dans la table enfant si un enregistrement correspondant est supprimé dans la table parente.
- ON UPDATE CASCADE : Met à jour automatiquement les enregistrements correspondants dans la table enfant si la valeur de la clé primaire dans la table parente est modifiée.

Exemple avec ON DELETE CASCADE CREATE TABLE employes ( id INT PRIMARY KEY, nom VARCHAR(50), prenom VARCHAR(50), departement_id INT, FOREIGN KEY (departement_id) REFERENCES departements(id) ON DELETE CASCADE );

Note : Avec ON DELETE CASCADE, la suppression d'un enregistrement dans la table departements entraînera automatiquement la suppression des enregistrements associés dans la table employes.
